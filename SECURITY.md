I do take the security of the projects i maintain seriously, sometimes i guess hahaha. While **Anti-YouTube-Shorts** is a client-side filter list and generally poses minimal inherent security risk, I appreciate any responsible disclosure of vulnerabilities that may affect users of this project.

### Supported Versions

I will actively try to maintains and supports the **latest stable release** of the filter list available in the `main` branch as long as i could.

### Reporting a Vulnerability

I personally request that any potential security vulnerabilities be reported privately, following the guidelines for **Responsible Disclosure**. Please **do not** report security bugs through public GitHub Issues or Discussions.

#### üìß How to Report

Please report security issues by emailing the maintainer directly:

* **Email:** `rekoy.official@protonmail.com`

#### **Information to Include:**

To help us quickly understand and fix the issue, please include the following details in your report:

1.  **A Clear Description:** Explain the vulnerability, its impact, and its severity.
2.  **Steps to Reproduce:** Provide a clear, step-by-step procedure or a proof-of-concept demonstrating the vulnerability.
3.  **Affected Component:** Specify which file or part of the filter list/repository is affected.
4.  **Configuration:** Include your browser and uBlock extension version.
